数据库  Database（DB）：是按照数据结构来组织，存储和管理数据的仓库。
	文件存储的缺点：
		1. 缺乏对数据的管理，数据不便修改；
		2. 不利于数据的分析和共享；
		3. 数据量巨大，大量的数据不能保存在文件中；

数据管理系统：Database Management System（DBMS），是数据库的管理软件。

关系型数据库：
	关系：描述两个元素间的关联或对应关系。

	常见的关系型数据库：
		Oracle（甲骨文，12c）
		MySQL（Sun公司收购，Oracle收购Sun公司，5.6）
		DB2（IBM）
		SQLServer（微软，2012）
		Sybase（Sybase，目前被SAP公司收购）

表：表是二维的，包含行和列，表的行是横排的（Row），也叫做记录（Record）；列是竖排的（Column），也叫做字段（Field）。
	一个关系型数据库是由多个数据表组成的，数据库的基本存储结构是数据表。
	表和表之间是有关联的。

结构化查询语言：SQL在关系型数据库上执行数据操作，检索，维护所使用的标准语言。可以用来查询数据，操作数据，定义数据，控制数据。

结构化查询语言包含如下5种：
	1. 数据定义语言：Data Definition Language（DDL）
	2. 数据操作语言：Data Manipulation Language（DML）
	3. 事务控制语言：Transation Control Language（TCL）
	4. 数据查询语言：Data Query Language（DQL）
	5. 数据控制语言：Data Control Language（DCL）

	1. 数据定义语言：Data Definition Language（DDL）
		用于建立，修改，删除数据库对象。
		包括：
			create：创建表或其他对象的结构。
				在数据库中大小写不敏感
				DESC：查看表结构
			alter：修改表或其他对象的结构。
				add：添加新字段在表的最后追加，不能添加到中间
				drop：drop()后面要跟删除的字段名
				modify：若表中有数据，类型尽量不要做修改，而且长度最好变长不要变短
			drop：删除表或其他对象的结构。
				drop代表删除该表中的数据及结构
			truncate：删除表数据，保留表结构。

	2. 数据操作语言：Data Manipulation Language（DML）
		用于改变数据表中的数据，和事务有关，执行完成后需要经过事务控制语句提交后才真正的将数据应用到数据表中。
		数据操作语言包含以下 3 种：
			insert：将数据插入到数据表中
			update：更新数据表中已存在的数据
			delete：删除数据表中的数据

	3. 事务控制语言：Transation Control Language（TCL）
		用于维护数据一致性的语句。
		包括：
			commit：提交，确认已经进行的数据改变。
			rollback：回滚，取消已经进行的数据改变。

	4. 数据查询语言：Data Query Language（DQL）
		用来查询所需要的数据
			select语句

	5. 数据控制语言：Data Control Language（DCL）
		用于执行权限的授予和收回的操作
		包括：
			grant：授予，用于给用户或角色授予权限。
			revoke：收回，用于收回用户或角色的已有权限。
			create user：创建用户

	default语句：给列指定默认值。
	not null：是一种约束条件，用于确保该字段不能为空，是一种非空约束。
		默认情况下，任何列都可以为空
		当某个字段被设置了非空约束，那么该字段就必须存在有效值
		当执行插入操作时，必须提供这个列的数据
		当执行更新操作时，不能给该值设置为空
	Number类型：
		表示数字类型，经常被定义成NUMBER(P,S)形式，其中p表示数字的总位数，s表示小数后面的位数。
		Number(6, 2) 表示这个数整数的最大值为4位 9999，小数为最大值为2为，取值最大为9999.99
		Number(P) 表示一个整数
		Number(4) 表示的最大整数为9999
	char类型：
		表示固定长度的字符类型
			经常被定义为char(n)形式，n表示占用的字节数，最大长度为2000字节。
			比如：char(20)表示最多可存储20个字节的字符串，并且占用的空间是固定的20个字节。
			每一个英文字母占用1个字节，每个中文字符按编码不同占用2-4个字节；
				gbk：2个字节
				UTF-8：3-4个字节
			char类型不设置长度，默认取值为1
	varchar(20)：
		用法同char
	varchar2类型：表示的变长的字符类型
		最大的取值范围为4000个字节
		定义的格式varchar2(20)
		表示最多可存储的长度为20个字节，根据所存的数据的长度，占用的空间是变化的，最大占用为20
		varchar2类型没有默认值，必须要设置长度。
	Long类型，是varchar2的加长版，存储的是变长字符串，最大能存2G的字符串数据。
		Long类型的使用有诸多限制，
			1. 每个表只能有一个long类型。
			2. 该字段不能作为主键。
			3. 不能建立索引。
			4. 不能出现在查询语句中。
	Clob类型，存储定长或变长字符串，最多能达4G的字符串数据。
	开发中，普通的字符串建议用varchar2，如果数据较大，建议使用Clob类型。
	Date类型
		用于定义日期时间的数据
		长度是7个字节
		默认的格式为：DD-MON-RR，比如：11-APR-79.表示79年7月11日
